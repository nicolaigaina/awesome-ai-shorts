# Contributing to Awesome AI Shorts

Thank you for contributing! This list aims to be the most comprehensive and useful resource for anyone creating short-form video content with AI.

## Guidelines

### Adding a Tool

- **Search first** — Check if the tool is already listed. Use Ctrl+F.
- **One tool per pull request** — Makes reviews easier.
- **Format:** `[Tool Name](https://url.com/) | Pricing | Platforms | One-line description of key differentiator`
- **Description rules:**
  - Start with a capital letter
  - End with a period (if a sentence) or no period (if a fragment)
  - Be specific about what makes this tool different — avoid generic "AI-powered video editor"
  - Keep it under 100 characters
- **Pricing:** Use `Free`, `Freemium`, `From $X/mo`, `$X one-time`, or `Open-source`
- **Placement:** Add the tool alphabetically within its category, unless there's a clear reason for different ordering.

### Adding a Tutorial or Resource

- Must be **high quality** — well-written, accurate, and genuinely useful
- Must be **publicly accessible** — no paywalled content without noting the paywall
- Must be **current** — published or updated within the last 18 months
- Include the source/author in the link text when relevant

### Quality Standards

We accept:
- Tools with an active user base or significant GitHub stars (100+)
- Open-source projects that are actively maintained (commits within last 6 months)
- Tutorials from reputable sources (official docs, well-known blogs, established publications)
- Statistics from primary research sources (not aggregator sites citing aggregator sites)

We do not accept:
- Tools that are clearly abandoned or non-functional
- Affiliate links or referral URLs
- Duplicate entries (same tool listed in multiple categories without reason)
- Self-promotional content without genuine value to readers
- Tools with no public pricing or information

### Suggesting a New Category

Open an issue first to discuss whether a new category is needed. Categories should have at least 3 entries to justify their existence.

## Pull Request Process

1. Fork the repository
2. Create a branch: `git checkout -b add-tool-name`
3. Make your changes to `README.md`
4. Ensure your addition follows the formatting guidelines above
5. Submit a pull request with a clear title: `Add [Tool Name]`
6. In the PR description, briefly explain why this tool belongs on the list

## Code of Conduct

Be respectful. This is a collaborative project. Disagreements about tool inclusion should be resolved through discussion, not hostility.

## Questions?

Open an issue if you're unsure about anything. We're happy to help!
